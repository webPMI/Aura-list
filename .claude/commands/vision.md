# /vision - App Vision and Goals

Reflect on and refine the app's purpose and direction.

## Instructions

This is a reflective command to ensure all improvements align with the app's core mission.

### Core Questions to Consider

1. **Who is this app for?**
   - Busy professionals?
   - Students?
   - Anyone managing daily tasks?
   - People with ADHD or executive function challenges?

2. **What problem does it solve?**
   - Remembering tasks
   - Prioritizing what matters
   - Feeling accomplished
   - Reducing mental load

3. **What makes it different?**
   - Simplicity over features?
   - Focus on wellbeing?
   - Beautiful design?
   - Specific workflow support?

4. **What should it NEVER become?**
   - Bloated with features
   - Anxiety-inducing
   - Complicated to use
   - A chore to open

### Vision Statement Template
```
[App Name] helps [target users] to [main benefit]
by providing [key differentiator]
so they can [desired outcome].

We believe that [core philosophy].
We will never [anti-pattern to avoid].
```

### Use This When
- Starting a major feature discussion
- Deciding between competing approaches
- Evaluating if a feature fits the app
- Onboarding new contributors

### Output
If `VISION.md` doesn't exist, create one with the user.
If it exists, review if current direction aligns with it.
